# Исчерпывающее руководство по настройке HP LaserJet (на примере P2055dn) в Linux (Pop!_OS / Ubuntu)

## 1. Суть проблемы: «Бесконечная печать кода и пустых листов»
Симптом: Принтер начинает безостановочно печатать листы, где вверху написан код вроде `%%@PJL ENTER LANGUAGE = PostScript` или `%!PS-Adobe-3.0`, а дальше идут пустые листы или иероглифы (помогает только выключение из розетки). Ошибка плавающая, часто возникает «через раз».

Причина: Драйвер PostScript (PS) в Linux иногда забывает отправить принтеру команду конца файла (Ctrl+D / EOF). Принтер зависает в режиме ожидания кода, и следующий отправленный документ воспринимает не как команду на печать, а как сырой текст, пытаясь распечатать мегабайты машинного кода. 

Решение: Полный отказ от драйвера PostScript и переход на стабильный аппаратный язык PCL3 (движок hpcups).

## 2. Подготовка и зачистка старых багов
Перед новой установкой нужно вычистить зависшую очередь и снести старые конфликтующие профили. Откройте терминал и выполните команды (каждую строку отдельно):

    # 1. Жестко отменяем все зависшие задания во всех принтерах
    cancel -a
    
    # 2. Ищем, как система назвала старые принтеры (ищите строки с 2055)
    lpstat -p
    
    # 3. Удаляем найденный профиль (замените ИМЯ_ПРИНТЕРА на то, что выдал шаг 2)
    sudo lpadmin -x ИМЯ_ПРИНТЕРА
    
    # 4. Устанавливаем официальный пакет драйверов HP
    sudo apt update
    sudo apt install -y hplip hplip-gui printer-driver-hpcups

## 3. Проблема «Невидимого USB»
Часто система Linux блокирует прямой доступ к USB-принтеру из-за фоновой службы ipp-usb (Driverless printing). Она захватывает порт и не дает установщику HP увидеть устройство.

Если принтер подключен по USB, но не определяется, выполните в терминале:

    sudo systemctl stop ipp-usb

После этого проверьте, видит ли система принтер аппаратно:
    
    lsusb
    
(В выводе должна быть строка вида `ID 03f0:5c17 HP, Inc LaserJet P2055 series`).

## 4. Идеальный алгоритм установки (Через веб-интерфейс CUPS)
Графические утилиты (типа hp-setup) часто конфликтуют с современными дисплейными серверами (Wayland) и правами root. Самый надежный способ установки — через браузер.

Шаг 4.1. Включаем веб-панель:
В терминале по очереди введите:

    sudo systemctl enable --now cups
    sudo cupsctl WebInterface=yes

Шаг 4.2. Добавляем принтер:
1. Откройте браузер и перейдите по адресу: http://localhost:631 (или http://127.0.0.1:631).
2. Перейдите на вкладку Administration (Администрирование) и нажмите Add Printer (Добавить принтер). Введите логин и пароль от вашей учетной записи Linux (системные).
3. В списке Local Printers выберите вариант, где есть слово HPLIP и серийный номер. 
   ПРАВИЛЬНО: HP LaserJet P2055dn USB S2AP28Q HPLIP
   НЕПРАВИЛЬНО: Обычный HP LaserJet P2055dn (без HPLIP).
4. Нажмите Continue. Задайте имя (без пробелов, например HP_P2055).
5. На странице выбора драйвера (Model) найдите огромный список для вашей модели. 
   ИЩЕМ И ВЫБИРАЕМ: HP LaserJet p2055dn pcl3, hpcups 3.23.12 (en)
   СТРОГО ИГНОРИРУЕМ: Любые строки, где в конце написано -ps или Postscript (recommended).
6. Нажмите Add Printer. На странице Defaults (параметры по умолчанию) нажмите Set Default Options. Принтер готов!

## 5. Важные нюансы и ответы на вопросы

Что значит приписка (en) в названии драйвера?
Она означает исключительно язык интерфейса системных настроек драйвера (размер бумаги, лотки и т.д. будут подписаны по-английски). Это никак не влияет на печать. Принтер без проблем распечатает текст на русском, японском или любом другом языке.

Двусторонняя печать и конфликты настроек
В веб-интерфейсе CUPS (в Defaults) можно задать базовый поворот страниц, например `sides=two-sided-short-edge` (переворот по короткому краю, идеален для книжной верстки A5->A4).
Если при печати из браузера (например, Firefox) вы тоже выберете «Переворот по короткому краю», конфликта не будет. Настройка приложения имеет приоритет и просто перезаписывает дефолтную команду на точно такую же. Двойного переворота не случится.

Можно ли печатать готовые PDF без браузера?
Да! Благодаря настроенным в CUPS дефолтам (duplex), вы можете печатать готовые книги для сшивки прямо из терминала одной командой, и принтер сам всё перевернет как надо:

    lp ИМЯ_ФАЙЛА_PRINT.pdf
