# AmneziaWG VPN Bypass for GitHub and Codeberg

Этот скрипт позволяет автоматически пускать трафик до GitHub и Codeberg в обход VPN-туннеля AmneziaWG (через вашего основного провайдера). Это полезно, если эти сервисы блокируют VPN-адреса или работают нестабильно.

## Как это работает
Скрипт добавляет специфические маршруты в основную таблицу маршрутизации системы:
1. Определяет ваш основной шлюз (роутер) и сетевой интерфейс.
2. Получает список актуальных IP-адресов GitHub через их API (`api.github.com/meta`).
3. Резолвит актуальные IP-адреса Codeberg.
4. Добавляет маршруты через `ip route add`, которые имеют приоритет над стандартным `0.0.0.0/0` в VPN.

## Установка

Для автоматической настройки запустите скрипт с флагом `setup`:

```bash
sudo ./amneziawg-bypass.sh setup
```

Это автоматически добавит строки `PostUp` и `PostDown` в ваш конфигурационный файл `/etc/amnezia/amneziawg/usa1.conf`.

Если ваш конфиг находится в другом месте:
```bash
sudo ./amneziawg-bypass.sh setup /путь/к/вашему/конфигу.conf
```

## Использование вручную

Вы также можете управлять маршрутами вручную без изменения конфига:

- **Добавить исключения**: `sudo ./amneziawg-bypass.sh add`
- **Удалить исключения**: `sudo ./amneziawg-bypass.sh del`

## Требования
- `bash`
- `curl` и `jq` (для получения IP-адресов GitHub)
- `python3` (для надежного определения IP-адресов Codeberg)
- Права `root` (через `sudo`)

## Проверка
После активации VPN проверьте маршрут до сервиса:

```bash
traceroute github.com
traceroute codeberg.org
```

Первым пунктом (`hop 1`) должен быть ваш локальный шлюз (например, `192.168.1.1`), а не VPN-сервер.
